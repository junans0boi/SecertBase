# MomentLoop 기획서

## 한 줄 정의

MomentLoop는 셋로그의 짧고 날것 같은 기록 방식을 커플/소그룹용 프라이빗 타임캡슐로 개선한 기능이다.

## 이름

서비스명: **MomentLoop**

선택 이유:

- Moment: 보정하지 않은 지금의 순간
- Loop: 하루 동안 반복해서 쌓이고, 밤에 다시 이어보는 루프
- Secret Base의 아케이드/아카이브 분위기와 어울리는 영어 이름이면서 한국어 화면에서는 "모먼트루프"로 자연스럽게 쓸 수 있다.

## 셋로그에서 가져올 장점

- 기록 단위가 짧아 부담이 적다.
- 편집하지 않은 원본성이 있어 SNS 피로감이 낮다.
- 친구/지인과 같은 시간대의 하루를 비교해 보는 재미가 있다.
- 하루가 끝났을 때 조각들이 하나의 흐름으로 이어지는 보상이 있다.

## 셋로그의 약점과 개선 방향

### 1. 알림 피로

문제:

- 매시간 알림은 초반에는 재미있지만, 바쁜 날에는 스트레스가 된다.

개선:

- 강제 매시간이 아니라 `하루 3회`, `식사 시간`, `랜덤 5회`, `직접 기록` 모드를 둔다.
- 상대와 둘 다 기록한 시간대만 특별하게 강조한다.

### 2. 보여주기 피로의 재발

문제:

- "날것"을 표방해도 점점 예쁘게 찍으려는 경쟁이 생길 수 있다.

개선:

- 좋아요 수, 공개 랭킹, 조회 수를 두지 않는다.
- 기록 카드에는 감정, 장소, 짧은 캡션만 남긴다.
- 편집 기능은 자르기/보정 없이 원본 유지 원칙을 둔다.

### 3. 놓친 순간의 허탈감

문제:

- 알림을 놓치면 하루 기록이 비어 보인다.

개선:

- 놓친 슬롯은 "그때 뭐 하고 있었어?" 텍스트 회고로 채울 수 있다.
- 놓친 기록도 실패가 아니라 하루 리듬의 일부로 표현한다.

### 4. 단순 영상 나열의 한계

문제:

- 짧은 영상 조각만 이어 붙이면 맥락이 약하다.

개선:

- 두 사람의 같은 시간대 기록을 나란히 보여준다.
- 하루 요약에 "동시에 바빴던 시간", "둘 다 쉬던 시간", "혼자만 남긴 순간"을 분리한다.
- 지도, Q&A, 챌린지와 연결해 아카이브 전체 맥락을 만든다.

## 핵심 기능

### Phase 1: 지금 순간 기록

- 아카이브 탭에서 MomentLoop 진입
- 내 현재 순간을 짧은 캡션으로 기록
- 오늘 기록 타임라인 표시
- 상대 기록과 내 기록을 시간순으로 함께 표시

### Phase 2: 원본 미디어 기록

- 카메라로 2-4초 영상 또는 즉석 사진 촬영
- 앨범 업로드보다 즉석 촬영 우선
- 원본 파일, 촬영 시각, 작성자, 캡션 저장

### Phase 3: 하루 루프

- 하루가 끝나면 둘의 기록을 시간순으로 묶은 루프 생성
- 같은 시간대 기록은 분할 화면으로 표시
- 빠르게 넘겨보는 `오늘의 루프` 화면 제공

### Phase 4: 알림 리듬

- 하루 기록 슬롯 자동 생성
- 커플별 기록 시간대 설정
- 놓친 슬롯 회고 입력

## MVP 범위

이번 개발 범위:

- 서버에 MomentLoop 기록 테이블 추가
- `GET /api/setlog`로 기록 조회
- `POST /api/setlog`로 텍스트 순간 기록 생성
- Flutter 아카이브의 셋로그 카드를 MomentLoop로 교체
- 실제 목록/작성 화면 제공
- 이미지 업로드 및 타임라인 이미지 표시
- 4초 영상 촬영/업로드 진입점
- 업로드된 영상 재생 카드

다음 개발 범위:

- 푸시 알림
- 하루 루프 생성
- 영상 합성

## 데이터 모델 초안

기록 단위는 `setlog_posts` 테이블에 저장한다.

- `id`: 기록 ID
- `couple_id`: 커플 ID
- `user_id`: 작성자 UserId
- `user_code`: 작성자 UserCode
- `media_type`: `text`, `image`, `video`
- `media_url`: 업로드 파일 경로
- `caption`: 짧은 기록
- `tags`: JSON 문자열
- `taken_at`: 기록 일자
- `captured_at`: 실제 기록 시각

## 제품 원칙

- 공개보다 친밀함을 우선한다.
- 예쁜 기록보다 진짜 기록을 우선한다.
- 실패한 기록도 하루의 일부로 취급한다.
- 게임 탭의 가벼움과 아카이브 탭의 따뜻함을 연결한다.
