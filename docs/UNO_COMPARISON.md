## 12. 2026-06-19 수정사항

### 모두내기 카드 규칙 수정

- 기존 구현은 “현재 바닥 색상과 같은 색 카드를 전부 내는 버튼”으로 동작했으나, 실제 규칙과 달라 제거했다.
- 덱에 색상별 `discard_all` 카드를 추가했다. 각 색상별로 2장씩 들어간다.
- `discard_all`은 와일드가 아니라 색상 카드다. 따라서 일반 색상/값 매칭 규칙으로 낼 수 있다.
- 예를 들어 파란색 모두내기 카드를 내면, 그 카드 1장과 함께 내 손패의 모든 파란색 카드가 자동으로 버려진다.
- 같이 버려진 카드 중 `draw2`, `skip`, `reverse`가 있으면 버려진 순서대로 효과를 적용한다.
- Flutter 카드 UI에는 숫자 자리 중앙에 겹친 카드 마크를 표시해 모두내기 카드임을 구분하게 했다.

### 2인전 Skip 턴 처리 고정

- 2인전에서 `skip`을 내면 상대 턴을 건너뛰므로 다시 낸 사람의 턴이 되어야 한다.
- 서버 엔진 테스트를 추가해 `skip -> getNextPlayer` 처리 후 현재 플레이어가 유지되는지 검증했다.
- 회귀 방지를 위해 `services/realtime-server/test/uno-engine.test.js`에 테스트를 추가했다.

### UNO / 잡기 버튼 통합

- UI에서 `UNO!` 버튼을 항상 하나만 표시하도록 정리했다.
- 내가 1장 남아 `unoPendingCall` 상태이면 같은 버튼이 `game:uno:call`을 보낸다.
- 상대가 1장 남아 `unoCatchable` 상태이면 같은 버튼이 `game:uno:catch`를 보낸다.
- 클라이언트에는 `SocketService.pressUnoButton()`을 추가해 UI가 서버 이벤트 종류를 직접 알 필요를 줄였다.
- `+4 도전`은 UNO 선언/잡기와 다른 룰 액션이라 별도 버튼으로 유지한다.

### 모바일 / 아이패드 / 가로모드 화면 개선

- 우노 화면을 공용 `GameScaffold`의 흰 앱바와 중앙 폭 제한에서 분리했다.
- 우노 전용 `Scaffold`를 사용해 어두운 보드 배경이 화면 전체를 채우게 했다.
- 뒤로가기, 제목, 메뉴는 공용 앱바 대신 어두운 상단 오버레이 HUD로 배치했다.
- 상태 문구도 흰 카드가 아니라 어두운 HUD 스타일로 변경했다.
- 보드에 `topInset`을 추가해 상단 오버레이와 상대 패/타이머가 겹치지 않게 했다.
- 앱 방향 제한을 세로 전용에서 세로/가로 모두 허용으로 변경했다.
- 모바일 세로, 모바일 가로, 태블릿 4:3/3:4에서 중앙 1:1 박스처럼 보이지 않도록 보드가 전체 가용 영역을 사용한다.

### 다중 기기 접속 처리

- 같은 계정이 PC와 휴대폰에서 동시에 접속하면 서버 presence가 중복 소켓 수만큼 증가하던 문제를 수정했다.
- `getPresence()`가 소켓 목록을 그대로 반환하지 않고 고유 `userId` 기준으로 반환하도록 변경했다.
- 결과적으로 같은 유저의 PC/휴대폰 동시 접속은 접속자 1명으로 계산된다.

### 배포 경로 확인 및 반영

- nginx 실제 root는 `/var/www/secretbase`임을 확인했다.
- 개발 확인용 `apps/secret_base_app/build/web` 또는 `localhost:8090`에만 빌드하면 실제 도메인에는 반영되지 않는다.
- release 빌드 후 `rsync -av --delete apps/secret_base_app/build/web/ /var/www/secretbase/`로 배포했다.
- nginx가 새 `index.html`과 `main.dart.js`를 서빙하는 것을 `Last-Modified` 헤더로 확인했다.

- `flutter build web --release`: 성공.
