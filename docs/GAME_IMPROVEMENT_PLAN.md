## [윷놀이] 버그 & 개선

### BUG-1: 대각선 위치(21~27)가 결승점으로 표시되는 문제 🔴

**파일**: `apps/secret_base_app/lib/ui/yut_board.dart` (line 154)

**원인**: `_getBoardPoint()` 함수의 첫 번째 조건이 `pos >= 20`으로 되어 있어서, 대각선 shortcut 위치인 21~27번이 모두 결승점 좌표(1.0, 1.0)로 반환됨.

```dart
// 현재 (버그 있음)
if (pos == 0 || pos >= 20) return const Offset(1.0, 1.0);

// 수정 후
if (pos == 0 || pos == 20) return const Offset(1.0, 1.0);
```

**증상**: 말이 꼭지점(5번 위치)에서 윷/모를 굴려 대각선 경로(21→22→23→...)에 진입하면, 화면에서 말이 즉시 결승점으로 점프해 보임.

**수정 난이도**: ⭐ 매우 쉬움 (1줄 변경)

---

### IMPROVE-1: 한게임 신윷놀이 꼭지점 진입 규칙 확인 필요 ⚠️

**현재 구현**: 말이 꼭지점(5, 10번) 위에 정확히 멈춰야 다음 이동 시 대각선 진입 가능. 꼭지점을 지나치는 경우(pass-through) 바깥 경로 유지.

**한게임 룰**:
- 꼭지점에 정확히 착지 → 다음 이동 시 대각선 선택 가능 ✅ (현재 구현과 동일)
- 꼭지점을 지나칠 경우 → 바깥 경로 계속 ✅ (현재 구현과 동일)

**결론**: 이동 로직(서버)은 정상. 버그는 UI 표시뿐.

## [UNO] 버그 & 개선

### BUG-2: +2/+4 카드를 내도 상대방이 카드를 받지 않음 🔴

**파일**: `services/realtime-server/src/socket.js` (game:uno:play 핸들러)

**원인**: `applyCardEffect()`에서 `drawStack`에 값이 쌓이지만, 현재 플레이어가 turn을 넘길 때 상대방이 강제로 drawStack을 소화해야 하는 제약이 없음. 즉, 상대방이 `game:uno:draw`를 누르지 않으면 아무 일도 안 일어남.

**수정 방향**:
- `game:uno:play` 시 `drawStack > 0`이면 방어 카드 이외의 카드 플레이를 서버에서 차단
- 상대방이 선택 UI를 통해 "방어" / "도전" / "받기"를 선택해야 함

### FEATURE-1: 카드 딜링 애니메이션 ✨

**현재**: 게임 시작 시 7장이 즉시 표시됨  
**목표**: 딜러(덱)에서 내 패로 카드가 한 장씩 날아오는 애니메이션

**구현 방법**:
- `UnoBoard`를 `StatefulWidget`으로 전환 (또는 내부에 Stateful 래퍼 추가)
- `gameId`가 null → non-null로 바뀌는 시점 감지
- 7장 카드를 순서대로 50~80ms 간격으로 `AnimatedPositioned`로 슬라이드 인
- 카드는 덱 위치에서 시작해 손패 위치로 이동

---

### FEATURE-2: +2/+4 누적 방어 시스템 (Draw Stack Chaining) ✨

**룰 정의 (모바일 UNO 표준)**:

#### +2 카드 규칙
| 상황 | 선택지 |
|------|--------|
| 상대가 +2를 냄 | ① 방어: 내 +2 카드 내기 (합산) ② 받기: 누적된 장수만큼 드로우 |

#### +4 카드 규칙
| 상황 | 선택지 |
|------|--------|
| 상대가 +4를 냄 | ① 방어: 내 +4 카드 내기 (합산) ② 도전: 적법성 확인 ③ 받기: 4장(+누적) 드로우 |

#### 누적 예시
```
A가 +4 냄 → drawStack: 4
B가 +4로 방어 → drawStack: 8
A가 +4로 재방어 → drawStack: 12
B가 방어 불가 → B가 12장 드로우
```

#### +4 도전 규칙
- **도전 성공** (공격자가 현재 색 카드를 가지고 있었음): 공격자가 **6장** 드로우, 도전자는 패널티 없음
- **도전 실패** (공격자가 정당하게 +4를 냄): 도전자가 **6장** 드로우 (4 + 2 추가 패널티) ← **확인 필요**
- 도전 안 함: 도전자가 4장(+누적) 드로우

---

**서버 변경 사항** (`socket.js`, `uno-engine.js`):

1. 게임 상태에 `drawStackType: 'draw2' | 'wild_draw4' | null` 추가
2. `game:uno:play` 핸들러에서 `drawStack > 0`일 때:
   - `drawStackType === 'draw2'`: +2 카드만 허용 (방어) 또는 거부 → 받기만 가능
   - `drawStackType === 'wild_draw4'`: +4 카드만 허용 (방어) 또는 거부 → 받기/도전 가능
3. 새 이벤트 `game:uno:challenge` 추가 (도전 처리)
4. `game:uno:draw` 수정: drawStack 있을 시 강제 소화

**클라이언트 변경 사항** (`socket_service.dart`, `uno_board.dart`):

1. `SocketService`에 `unoDrawStack: int`, `unoDrawStackType: String?` 추가
2. `UnoBoard`에 방어/도전/받기 선택 바텀시트/다이얼로그 추가
3. drawStack 표시 UI (현재 얼마나 쌓였는지 가시적으로)

---

### FEATURE-3: 특수 카드 시각 이펙트 ✨

**대상 카드**: SKIP, Reverse, Draw2(+2), Wild Draw4(+4)

**구현 방법**:
- `SocketService`에 `unoLastSpecialCard: Map?`, `unoSpecialCardBy: String?` 추가
- `game:uno:played` 이벤트 수신 시 특수 카드 여부 감지
- `UnoBoard`에 이펙트 오버레이:
  - **SKIP**: 빨간 "SKIP!" 텍스트 + 원형 파동 애니메이션
  - **Reverse**: 방향 화살표 회전 애니메이션
  - **+2**: 카드 2장이 날아오는 애니메이션
  - **+4**: 카드 4장 + 색상 변경 강조 이펙트
- 약 1.5초 후 자동 소거
