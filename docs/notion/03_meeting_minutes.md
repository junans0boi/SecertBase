# 03. [회의록] 비밀기지(Secret Base) 가상 기획 및 개발 회의 기록

> [!NOTE]
> 본 문서는 프로젝트의 킥오프 단계부터 배포 및 운영 준비 단계에 이르기까지 진행된 핵심 의사결정 회의를 재구성한 가상의 회의록이다. 기획 의도 및 구현상의 합의 과정을 타임라인별로 자세히 수록하여 노션(Notion)의 회의 아카이브 템플릿에 맞춤형으로 사용할 수 있다.

---

## 1. 제 1회차 회의: 프로젝트 킥오프 및 요구사항 정의
* **일시**: 2026-06-01 (14:00 - 15:30)
* **참석자**: 준쨩(기획/클라이언트 검토), AI Copilot(설계 및 풀스택 개발)
* **안건**: 프로젝트 기획 방향성 수립 및 기술 스택 확정

### 논의 내용
1. **서비스 콘셉트**:
   * 인스타그램 같은 공개형 SNS의 피로도를 덜어내기 위해 **'오직 우리 둘만 접속할 수 있는 프라이빗 웹/앱 서비스'**로 방향 설정.
   * "술자리 및 가벼운 커뮤니케이션용 실시간 게임" + "커플 일상 기록 보관소(아카이브)" 두 개의 축으로 설계.
2. **기술 스택 선정**:
   * **프론트엔드**: 웹과 안드로이드 하이브리드 지원이 뛰어난 **Flutter** 채택.
   * **백엔드**: 실시간 양방향 통신 성능이 우수하고 생산성이 빠른 **Node.js + Socket.IO + Express** 선정.
   * **메모리/세션 캐시**: 게임 상태 캐싱 및 방 접속 세션을 초고속으로 처리하기 위한 **Redis** 도입.
   * **영구 저장 데이터**: 회원 정보 및 커플 아카이브 기록 처리를 위한 **MariaDB(MySQL 규격)** 선정.
3. **일정/마일스톤 수립**:
   * 2.5개월 단기 스프린트로 MVP 및 실배포 완료 목표 수립.

### 결정 사항
* [x] 프로젝트명 **"비밀기지 (Secret Base)"** 확정.
* [x] Monorepo 구조로 프로젝트 개발 리포지토리 초기화 결정.
* [x] 파트너 연결을 위한 6자리 난수 'UserCode' 기반 인증 플로우 도입.

## 2. 제 2회차 회의: 아케이드 존 실시간 게임 엔진 및 동기화 설계
* **일시**: 2026-06-08 (16:00 - 17:30)
* **참석자**: 준쨩, AI Copilot
* **안건**: 실시간 미니게임 8종 명세 확정 및 소켓/물리 엔진 동기화 방식 설계

### 논의 내용
1. **실시간 미니게임 라인업 확정**:
   * 1회성 간이 게임: 주사위, 룰렛, 가위바위보, 텔레파시, 해적 룰렛.
   * 본격 턴제 코어 게임: **윷놀이(Yut)**, **UNO 카드**, **폭탄 돌리기**.
2. **윷놀이 및 UNO 세부 동기화 설계**:
   * 클라이언트의 치팅(조작)을 막기 위해 윷가락 계산, 말 이동 가능 루트 탐색, 우노 카드 드로우/셔플 로직 등 **모든 핵심 게임 로직은 서버 엔진(JS 파일)에서 독점 계산**하기로 함.
   * 클라이언트는 서버가 브로드캐스트하는 GameState를 받아 화면을 렌더링하는 역할로 한정.
3. **UNO 리액션 추가 아이디어**:
   * 두 명이서만 카드 게임을 진행하므로 밋밋할 수 있음. 케이크, 커피, 피자 등을 던질 수 있는 사운드 연동형 기프트 리액션 추가 제안.

### 결정 사항
* [x] UNO 엔진에 클래식 룰(일반 우노)과 고와일드 룰(공격카드 누적 방어, `discard_all` 특수 카드 제공)의 2가지 모드 탑재 결정.
* [x] 윷놀이 말 이동 루트(shortcut) 판정 연산을 `yut-engine.js`에 전면 위임.
* [x] 소켓 서버 연결 오류 및 예기치 못한 레이턴시에 대응할 수 있도록 Ping/Pong 실시간 지연 체크 UX 설계.

## 3. 제 3회차 회의: 아카이브 기능 고도화 및 리텐션 루프 기획
* **일시**: 2026-06-15 (14:00 - 15:30)
* **참석자**: 준쨩, AI Copilot
* **안건**: 아카이브 존 명세 세분화 및 리텐션(재방문율)을 유도하기 위한 일일 루프 기획

### 논의 내용
1. **기록의 피로감 줄이기**:
   * 매시간 인위적인 알림을 줘 강제 기록하게 하는 서비스들의 단점을 보완하기 위해 'MomentLoop(모먼트루프)' 제안.
   * 영상/사진 등의 비주얼 기록은 무보정 즉석 촬영을 원칙으로 하며, 하루 3회 또는 연인이 기록하고 싶을 때 직접 작성하는 모드를 병행 지원.
2. **Q&A 답변 마스킹 UX**:
   * 커플 전용 데일리 문답 진행 시, 내가 답변을 입력하기 전까지 상대방의 답변 내용을 감춰야만 솔직하고 재미있는 문답이 이루어진다는 점에 전원 합의.
3. **커플 스트릭(Streak) 도입**:
   * 개별 스트릭이 아니라 커플 공동의 날짜 스트릭을 매겨, 둘 다 적극적으로 하루의 미션을 달성하도록 보상 체계 마련.

### 결정 사항
* [x] 아카이브의 5대 핵심 메뉴 확정: MomentLoop, 비밀 지도, 10시 Q&A, 목표 챌린지, 주크박스.
* [x] 주크박스 업로드 용량 제한을 곡당 10MB로 백엔드에서 제어.
* [x] 데이터베이스 테이블을 설계하여 REST API 개발 프로세스로 즉시 전환.

## 4. 제 4회차 회의: 개발 중간 점검 및 예외복구/CORS 트러블슈팅
* **일시**: 2026-06-20 (15:00 - 16:30)
* **참석자**: 준쨩, AI Copilot
* **안건**: 윷놀이/UNO UI 피드백 및 네트워크 예외 처리(CORS, Reconnect) 방안 마련

### 논의 내용
1. **재접속 시 상태 유실 문제**:
   * 플레이 중 지하철이나 엘리베이터 등 음영 구역에서 일시적으로 소켓이 끊기면 판 전체가 날아가는 문제가 식별됨.
   * **해결책**: 백엔드에서 게임 상태가 변할 때마다 Redis 캐시에 백업하고, 클라이언트가 재접속 시 자동으로 `session:restore`를 호출하여 이전 상태를 복원하도록 프로세스 마련.
2. **로컬 개발환경 CORS 404 에러**:
   * Flutter Web 디버그 환경 포트가 분리되며 API 통신에 실패하는 문제를 확인.
   * `server_config.dart`에 호스트 판별 가드를 추가하여 로컬과 실서버 접속 주소를 동적으로 결정하도록 조치함.
3. **데이터베이스 마이그레이션**:
   * 새로 배포된 서버에 신규 테이블이 수동 반영되지 않아 에러가 나던 문제를, 백엔드 라우터 기동 시 테이블 누락 여부를 판단하여 자동으로 DB DDL을 수행하는 `ensureTables()` 헬퍼를 심는 것으로 자동화함.

### 결정 사항
* [x] Redis 캐시 연동을 Yut, UNO, Bomb 엔진에 전체 구현 완료.
* [x] Flutter 웹 클라이언트 로컬 디바이스용 포트 및 CORS 화이트리스트 튜닝 적용.
* [x] 윷놀이/UNO 기본 레이아웃 구성 및 REST API 연결 테스트 통과.

## 5. 제 5회차 회의: 최종 검수 및 Nginx/PM2 배포 전략 수립
* **일시**: 2026-06-22 (11:00 - 12:30)
* **참석자**: 준쨩, AI Copilot
* **안건**: 도메인 연동 및 프로덕션 실환경 배포 완료 여부 체크

### 논의 내용
1. **도메인 연결 및 SSL 보안화**:
   * 무료 도메인 대행 서비스를 활용하여 `secertbase.kro.kr`로 도메인 타겟 지정.
   * Nginx 환경설정을 통해 80(HTTP) 트래픽을 443(HTTPS)으로 리디렉션 처리하고, API 요청(`/api`)과 웹소켓(`/socket.io`) 경로를 노드 포트로 정상 프록싱하고 있는 점 검증.
2. **서버 무중단 기동 및 모니터링**:
   * 백그라운드 구동을 위한 `pm2` 서비스를 상시 운영 상태로 등록하고, 빌드/배포 프로세스를 쉘 스크립트(`deploy_server.sh`)로 패키징하여 자동화함.
3. **차기 과제 및 고도화 항목**:
   * 윷놀이 윷 던지는 물리 연산 연동 강화 및 카드 선택 드래그 앤 드롭 폴리싱.
   * 푸시 알림 수신을 위한 Firebase Cloud Messaging(FCM) 도입 검토.

### 결정 사항
* [x] 실서버 배포 상태 보장 (Health Check 엔드포인트 `/health` 가동 확인).
* [x] 실 운영 웹 주소(`https://secertbase.kro.kr`) 및 DB 터널링 경로 구성 완료.
* [x] 1차 릴리즈 본의 기획/개발 일상 정리 완료 선언 및 노션 백업 추진.
